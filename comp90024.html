3 Lecture 3: Overview of Distributed and parallel computing

SPARTAN Amdahl's law

3.1 Approaches for parallelism:

  1. Explicit vs implicit architecture Implicit :- parallelising compiler Explicit - Programmer is responsible for parallelism. (SPARTAN)
  2. Hardware
    1. Threading CPU
    2. Multi-Core
    3. Types of Multiprocessing:
      1. SMP- Symmetric multiprocessing ( all processors ar peers)
      2. NUMA - Non uniform memory access ( master/slave)
  3. OS
    1. Most modern multi-core OS supports parallelism.
    2. Compute parallelism - using Processes / Threads
    3. Data parallelism : Using caching
  4. Software/Applications
    1. Most languages supports parallelism: Threads, threadpools, locks, semaphores
    2. Key issues: Deadlocks/livelocks

3.2 Advantages of Parallelism:

  1. Increased throughput
  2. Economy of scale:
  3. Increased reliability

3.3 MPI

  1. HT Condor
    • Offers job queuing, scheduling policies,priority, resource management
    • Allows to harvest CPU power from otherwise idle desktop
    • No need to share filesystems
    • can work across organizational boundaries

3.4 Erronious assumptions of Distributed system

  1. Network reliability
  2. Latency is zero
  3. Bandwidth is infinite
  4. network is secure
  5. Topology doesnt change
  6. There is one admin
  7. Transport cost is zero
  8. Network is homogenous
  9. Time is ubiqutous

3.5 Parallel programming paradigms

  1. Master worker/slave model
  2. Single program multiple data -> Map reduce
  3. Data pipelining -> multiple stages of execution
  4. Divide and conquer
  5. Speculative parallelism: when others fail. Uses "look ahead" execution

4 Lecture 4: SPARTAN HPC system

4.1 High performance computing characteristics

  1. Above average performance
  2. Clustered computing: multiple computers serve single resource
  3. imporves performace and increase redundancy
  4. Effecient, economical and scalable
  5. Easy to parallelise
  6. Linux based: scalable, resource effecient
  7. multi-process,multi-core,multi-thread,multi-node
  8. Limitation: speedup = SerialTime/ParallelTime
  9. Parallelism required synchronization for correctness.
  10. Reducing latency, increasing throughput, and reducing CPU power consumption
  11. Amdahl's law -> Speedup is limited by the serial portion of work
  12. Gustafson and Barsis method -> Parallelism enable computations on very large data sets in fixed amount of time.
